# Contributing

Thanks for your interest in contributing.

## Before opening a PR
- Open an issue first for major changes or new features.
- For small fixes, feel free to open a PR directly.

## Development
1. Clone the repository
2. Install the required dependencies
3. Run the project locally
4. Run formatting, linting, and tests before submitting

## Code style
- Keep changes focused and small
- Prefer readable and well-structured code
- Add or update tests when relevant
- Update the README if behavior or usage changes

## Commit messages
Use clear commit messages describing the intent of the change.

Examples:
- `fix: handle websocket reconnect correctly`
- `feat: add checksum validation`
- `docs: improve setup instructions`

## Pull requests
Please include:
- a short summary of the change
- why the change is needed
- testing notes
- screenshots/logs if useful
